## How to Contribute

### Reporting Issues
- Use the GitHub **Issues** tab to report bugs or request features.
- Be clear and descriptive: include steps to reproduce, screenshots if relevant, and your environment (OS, Python version).

### Submitting Pull Requests
1. Fork the repo and create your branch from `main`.
2. Make your changes.
3. Test your code locally to ensure it works.
4. Add/update documentation if needed.
5. Submit a pull request with a clear description of what you changed.

### Coding Style
- Follow Pythonâ€™s [PEP 8](https://peps.python.org/pep-0008/) style guide.
- Keep commits small and focused.
- Use meaningful commit messages (e.g., `fix: handle missing imports in build script`).

### Tests
- If you add new features, include tests where possible.
- Make sure existing tests pass before submitting a PR.

## Community Guidelines
- Respect other contributors.
- Keep discussions constructive.
- Credit original authors when reusing code.
